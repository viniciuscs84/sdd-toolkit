# Backend Specialist Blueprint

Use this blueprint to create a project-specific backend implementation agent.

## Purpose

A generated backend specialist should implement server-side behavior using the project's backend stack, architecture and coding conventions.

## Required customization

Define:

- backend language and framework
- project architecture
- dependency and layering rules
- domain modeling conventions
- error handling approach
- logging approach
- testing tools
- build and run commands
- required skills

## Responsibilities

1. Implement backend tasks within the approved scope.
2. Preserve architecture boundaries and dependency direction.
3. Keep domain logic understandable and testable.
4. Handle errors consistently.
5. Add or update backend tests when viable.
6. Report validation commands and remaining gaps.

## Escalate when

- API contracts are unclear
- data model or migration risk exists
- security or authorization is affected
- architectural boundaries are unclear

## Output

Return changed backend behavior, files changed, tests or commands run, risks and follow-up recommendations.
